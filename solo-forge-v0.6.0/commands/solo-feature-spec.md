# 기능 스펙 작성

새로운 기능에 대한 명세 문서를 생성합니다.

## 사용법

```
/feature-spec
```

## 수집 정보

PM(개발자)에게 다음을 물어봅니다:

1. **기능명** (예: "사용자 로그인")
2. **대상 사용자** (예: "모든 미가입 사용자")
3. **핵심 요구사항** (3~5개 항목)
4. **우선순위** (필수/높음/중간/낮음)

## 스펙 문서 생성

저장 경로: `docs/02_기획/spec_{기능코드}.md`

기능코드: 기능명의 로마자 표기 축약 (예: `spec_login.md`)

템플릿:
```markdown
# 기능 스펙: {기능명}

## 개요

{기능명}은 {대상 사용자}가 {핵심 목표}를 달성하기 위한 기능입니다.

**우선순위**: {필수/높음/중간/낮음}

## 사용자 스토리

### US-001: {역할}로서 {목표}하고 싶다
- **액터**: {사용자 타입}
- **시나리오**: {구체적 상황}
- **기대 결과**: {원하는 결과}
- **AC**:
  - [ ] 조건 1
  - [ ] 조건 2

### US-002: ...

## 기능 요구사항

| ID | 요구사항 | 설명 |
|----|----------|------|
| FR-001 | {요구사항 1} | {상세 설명} |
| FR-002 | {요구사항 2} | {상세 설명} |

## 화면 목록

| 화면명 | 설명 | Figma 링크 |
|--------|------|-----------|
| {화면 1} | {용도} | [링크 또는 "미작성"] |
| {화면 2} | {용도} | [링크 또는 "미작성"] |

## 데이터 모델

### {엔티티 1}
```
{
  "id": "string",
  "field1": "type",
  "field2": "type"
}
```

### {엔티티 2}
```
...
```

## API 명세

### POST /api/{기능}
- **설명**: {목적}
- **인증**: {필요/불필요}
- **요청**:
  ```json
  {
    "param1": "type",
    "param2": "type"
  }
  ```
- **응답**:
  ```json
  {
    "success": "boolean",
    "data": {}
  }
  ```

## 예외 케이스

- {예외 1}: 처리 방식
- {예외 2}: 처리 방식

## 의존성

- [의존 기능 1]
- [의존 기능 2]

## 검토 상태

- [ ] PM 검토
- [ ] 설계자 검토
- [ ] 개발자 검토

---

작성일: YYYY-MM-DD
최종 수정: YYYY-MM-DD
버전: 1.0
```

## 완료 보고

```
기능 스펙 작성 완료.

**기능**: {기능명}
**저장 위치**: docs/02_기획/spec_{기능코드}.md
**주요 내용**:
  - 사용자 스토리: X건
  - 기능 요구사항: Y건
  - 화면 목록: Z건

이 스펙을 바탕으로 설계/개발을 진행하시면 됩니다.
```

## 체크리스트

스펙 작성 전 확인:
- [ ] 기능명과 목표가 명확한가?
- [ ] 대상 사용자가 정의되었는가?
- [ ] 핵심 요구사항이 우선순위와 함께 정리되었는가?
- [ ] Figma Make 사용 여부 확인했는가?
