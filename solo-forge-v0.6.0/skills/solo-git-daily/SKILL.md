# git 일일 다이제스트 (solo-git-daily)

git 커밋 이력을 읽어(**읽기 전용 — fetch/log만, 자동 커밋·`add` 안 함**) 프로젝트 진행 로그에 날짜별로 반영한다. "무엇을 언제 했나"의 기록을 손으로 안 써도 남긴다. 세션 로그(무엇을 했다)와 상보적이다.

## 실행 순서

**1단계 — 기간 결정**
`$ARGUMENTS`로 기간을 받는다(없으면 어제 00:00 ~ 오늘 00:00). 예: `어제`, `오늘`, `2026-07-10..2026-07-13`.

**2단계 — 커밋 수집 (읽기 전용)**
```bash
git log --since="<start>" --until="<end>" --pretty=format:'%h %ad %s' --date=short --no-merges
```
원격 반영분까지 보려면 먼저 `git fetch --all --prune`(선택). **여기서 `git add`·`git commit`·`push`를 하지 않는다.**

**3단계 — 프리픽스 파싱 (선택)**
커밋 메시지 접두를 파악해 분류한다. 있으면 활용, 없으면 그냥 목록.
- Conventional Commits: `feat(scope):` · `fix:` · `refactor:` · `docs:` · `test:` …
- 대괄호 태그: `[TSK-12]` · `[auth]` 등 → 추적키(`solo-consistency-check`의 키)와 연결.

**4단계 — 진행 로그 append (멱등)**
`docs/00_관리/진행로그.md`(없으면 생성)에 날짜 섹션으로 추가한다.
- **멱등 규칙**: append 전에 그 커밋 SHA(`%h`)가 로그에 이미 있는지 grep한다. 이미 있으면 건너뛴다 → 여러 번 돌려도 중복되지 않는다.
- 형식:
```
## 2026-07-13
- `a1b2c3d` feat(auth): JWT 발급 엔드포인트 [TSK-12]
- `d4e5f6a` fix: 만료 토큰 처리 [TSK-13]
```

**5단계 — 요약 보고**
기간·커밋 수·주요 변경을 한두 줄로 요약한다. 프리픽스 분류가 있으면 유형별 카운트도.

## ⚠️ 커밋 안전 프로토콜 (L-24 — 사용자가 이후 커밋할 때)

이 스킬은 자동 커밋하지 않지만, **사용자가 이 프로젝트에서 커밋할 때** 아래를 지키도록 안내한다. 특히 레포가 **iCloud Drive / 클라우드 동기 폴더**에 있으면 동기화가 `.git/index`를 건드려 **유령 대량 삭제**가 생길 수 있다:

1. **커밋 전 항상 `git status` 확인** — 무엇이 staged 됐는지 눈으로 본다.
2. **타깃 `git add <파일>`만** — 광범위 `git add -A`/`git add .`는 피한다(의도치 않은 삭제·추가를 쓸어담음).
3. **예상외 삭제(`D`) 감지 시 커밋 중단** — status에 만들지 않은 삭제가 보이면 커밋하지 말고 조사한다(`git reset`으로 인덱스만 비파괴 복구; 워킹트리·HEAD·origin은 대개 무손실).
4. **진짜 백업은 `git push`(원격)** — 클라우드 동기화는 백업이 아니라 위험원으로 취급한다.

> 이 프로토콜은 실측 사고에서 도출됐다: 클라우드 동기가 인덱스만 손상시켜 352파일 유령 삭제가 staged 됐고, `git add -A`였다면 대량 삭제로 커밋될 뻔했다. `git status` + 타깃 add가 그것을 막는다.

## 원칙

- git 쪽은 **읽기 전용**이다(fetch/log). 로그 반영은 마크다운 파일 append(사용자 확인 후).
- 멱등을 SHA grep으로 보장한다 — 재실행이 안전하다.
