# QA 시나리오 템플릿 가이드

> qa-scenario-builder 스킬에서 참조하는 테스트 시나리오 작성 가이드입니다.

## TC ID 체계

```
TC-{기능코드}-{순번}
```

**기능코드 규칙:**
- 영문 대문자 2~5자
- 기능 영역을 축약 (예: AUTH, DASH, NOTI, PROJ, USER)
- 프로젝트 내에서 고유

**예시:**
- `TC-AUTH-001`: 인증 기능 첫 번째 테스트 케이스
- `TC-DASH-015`: 대시보드 기능 15번째 테스트 케이스

## 시나리오 템플릿

### 마크다운 형식 (기본)

```markdown
# QA 테스트 시나리오 — {기능명}

**작성일**: YYYY-MM-DD
**작성자**: PL 준호
**기준 문서**: {feature spec 파일명 또는 D-XX}
**버전**: v0.1

## 요약

| 항목 | 값 |
|------|-----|
| 총 TC 수 | N |
| P1 (Critical) | N |
| P2 (Major) | N |
| P3 (Minor) | N |
| P4 (Low) | N |

## 테스트 케이스

### {기능 그룹 1}

| TC ID | 시나리오명 | 우선순위 | 사전 조건 | 테스트 단계 | 기대 결과 | 실제 결과 | 상태 |
|-------|-----------|---------|-----------|------------|-----------|-----------|------|
| TC-XXX-001 | {시나리오명} | P1 | {사전 조건} | 1. {단계1}<br>2. {단계2} | {기대 결과} | — | ⬜ |
| TC-XXX-002 | {시나리오명} | P2 | {사전 조건} | 1. {단계1} | {기대 결과} | — | ⬜ |

### {기능 그룹 2 — 예외 케이스}

| TC ID | 시나리오명 | 우선순위 | 사전 조건 | 테스트 단계 | 기대 결과 | 실제 결과 | 상태 |
|-------|-----------|---------|-----------|------------|-----------|-----------|------|
| TC-XXX-E01 | {예외 시나리오} | P2 | {사전 조건} | 1. {단계} | {에러 처리 기대} | — | ⬜ |

## 커버리지 매핑

| 요구사항 ID | 요구사항 | 관련 TC |
|------------|---------|---------|
| REQ-001 | {요구사항} | TC-XXX-001, TC-XXX-002 |
| REQ-002 | {요구사항} | TC-XXX-003 |
```

### 상태 값

| 상태 | 기호 | 의미 |
|------|------|------|
| 미실행 | ⬜ | 아직 테스트 미실행 |
| 통과 | ✅ PASS | 기대 결과와 일치 |
| 실패 | ❌ FAIL | 기대 결과와 불일치 |
| 건너뜀 | ⏭ SKIP | 환경/조건 미충족으로 건너뜀 |
| 차단됨 | 🚫 BLOCKED | 선행 조건 미해결로 실행 불가 |

### 우선순위 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| P1 Critical | 핵심 경로, 비즈니스 필수 | 로그인, 결제, 데이터 저장 |
| P2 Major | 주요 기능, 사용 빈도 높음 | 검색, 필터, 목록 조회 |
| P3 Minor | 보조 기능, 엣지 케이스 | 정렬, 페이지네이션, 툴팁 |
| P4 Low | UI 미세 조정, 선호 사항 | 폰트 크기, 간격, 색상 미세 차이 |

## Google Sheets 연동 시

Google Sheets MCP가 연결된 경우, 마크다운 대신 시트에 직접 작성합니다.

**시트 구조:**
- Tab명: `QA_{기능코드}` (예: QA_AUTH)
- Row 1: 헤더 (TC ID, 시나리오명, 우선순위, 사전 조건, 테스트 단계, 기대 결과, 실제 결과, 상태, 비고)
- Row 2~: 테스트 케이스
- 마지막 행: 요약 (총 TC, PASS/FAIL 수, 통과율)

**MCP 미연결 시:** 마크다운 파일로 `docs/05_검증/`에 저장합니다.
