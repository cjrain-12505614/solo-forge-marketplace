# MCP 도구 인벤토리 (solo-mcp-inventory)

지금 이 세션에 어떤 MCP 서버·도구가 연결돼 있는지 한눈에 보여주고, 더 필요하면 어떻게 켜는지 안내한다. 어떤 작업을 자동화할 수 있는지 파악할 때 유용하다.

## 보고 절차

**1단계 — 현재 세션 활성 MCP 인벤토리**
지금 사용 가능한 도구 중 `mcp__<서버>__<도구>` 형태를 **서버별로 묶어** 나열한다.
- 서버명 · 도구 수 · 대표 도구 몇 개
- 인증이 필요해 아직 못 쓰는 서버가 있으면 "인증 대기"로 표시(연결은 됐지만 OAuth 등 미완).

**2단계 — 활성화 안내 (더 켜려면)**
- **claude.ai 커넥터**: claude.ai 설정 → 커넥터에서 연결/인증.
- **로컬/CLI MCP 서버**: 인터랙티브 세션에서 `claude mcp` 또는 `/mcp`로 추가·인증.
- 비인터랙티브(이 세션)에서는 OAuth 흐름을 돌릴 수 없으니, 인증이 필요한 서버는 위 경로로 사용자가 직접 연결해야 한다고 안내한다.

## 보고 본문 골격

```
## MCP 인벤토리 (현재 세션)

### 활성 (N개 서버)
- <서버A> — 도구 12개 (예: get_x, list_y, …)
- <서버B> — 도구 5개 (…)

### 인증 대기 / 미활성
- <서버C> — 인증 필요 (claude.ai 커넥터 또는 /mcp)

### 더 켜려면
- claude.ai 커넥터 설정 / `claude mcp` / `/mcp`
```

## 원칙

- **지금 세션의 실제 도구를 근거로 보고**한다. 설치돼 있을 "것 같은" 도구를 추측해 나열하지 않는다.
- 특정 도구 목록을 하드코딩하지 않는다 — 세션마다 붙는 MCP가 다르므로 실측 인벤토리만 낸다.
