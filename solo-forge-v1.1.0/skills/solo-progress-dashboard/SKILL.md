# 범용 진행 대시보드 (solo-progress-dashboard)

> **계보**: douzone-forge `dz-progress-dashboard` v0.2.0(실측 검증 완료)에서 이식. 특정 워크스페이스 의존을 제거하고 solo-forge 계약(CLAUDE.md · `docs/00_관리/`)에 맞게 범용화했다.

> **용어 풀이**: 대시보드(진행을 한 화면에 그려 보여주는 현황판) · 마커(HTML에 심어두는 `<!-- 표식 -->`, 갱신 위치를 가리킴) · 부분 치환(전체를 다시 만들지 않고 표시 부분만 바꿈) · 멱등(같은 입력이면 여러 번 돌려도 결과 같음) · 순수 칸 비율(완료 칸 ÷ 전체 칸) · 표시본(브라우저가 읽는 사본) · PL(본 스킬에서는 프로젝트를 실행하는 에이전트, 즉 Claude)

## 0. 한 줄 정의

복잡하거나 단계가 많은 작업을 **시작할 때 대시보드를 만들어 보여주고, 단계가 바뀔 때마다 자동으로 갱신한다.** 특정 프로젝트 전용이 아니라 **모든 다단계 작업에 쓰는 범용 도구**다.

## 1. 트리거 / 발동

**사용자 호출** (한글 키워드):
- "대시보드 만들어줘" · "진행 대시보드" · "대시보드 갱신·반영" · "진행 가시화" · "대시보드 보여줘"

**자동 발동** (PL 판단 — §6 신호 3개 이상):
- 다단계·다영역 작업을 감지하면 PL이 먼저 **"대시보드로 추적하겠습니다" 1줄 고지** 후 생성 (자율이되 투명 — 사용자가 원치 않으면 끔)

**상위 대시보드 우선** (생성 전 필수 점검):
- 이미 상위(프로젝트·사이클) 대시보드가 있으면 **하위 작업 대시보드를 새로 만들지 않는다.** 상위 대시보드의 해당 항목 상태만 변경해 일관을 유지한다.
- 발동 판단 전 상위 대시보드 존재 여부(§12 보관 위치)를 먼저 확인한다. 하위 생성은 상위가 없을 때만.

## 2. 영역 분리 (기존 현황 스킬과 — 중복 회피)

| 스킬 | 영역 | 단위 |
|---|---|---|
| `solo-status-reporter` | 프로젝트 종합 현황 (텍스트 보고서) | 프로젝트 전체 |
| **`solo-progress-dashboard`** (본 스킬) | **단계 진행 HTML 대시보드 (자동 생성·갱신)** | **개별 작업 1건** |

→ status-reporter = "프로젝트가 지금 전체 어디"(스냅샷·텍스트). 본 스킬 = "이 작업이 어디까지"(작업 1건의 단계 흐름·시각·자동 갱신).

## 3. 동작 흐름 (5 단계)

```
작업 시작
 → ① 복잡도 판단 (§6 신호 3개 이상)
 → ② 복잡하면 작업 유형 선택 (§4 카탈로그) + "대시보드로 추적합니다" 고지
 → ③ 공통 코어(§5) + 유형 변주로 대시보드 생성 → 표시
 → ④ 단계 전환마다 자동 갱신 (완전 자동 — 파일만 수정, §9)
 → ⑤ 작업 종료 시 보관 (§12)
```

## 4. 작업 유형 카탈로그 (6 종 — PL이 고름, 무한 변형 아님)

| 유형 | 예시 | 유형 특화 영역 (공통 코어에 더함) |
|---|---|---|
| 분석형 | 시장조사·벤치마킹·실험 분석 | 조사 단계 + 출처 격자 + 검증 상태 |
| 산출물 파이프라인형 | md→html→pdf→pptx | 단계 흐름 띠 + 산출물별 완료 체크 |
| 설계형 | 요구사항→IA→화면설계 | 산출물 단계 + 검토 게이트(G-입력·G-출력) |
| 개발형 | TDD 구현 | RED→GREEN→REFACTOR + 빌드·테스트 상태 |
| 표준화·다영역형 | 다영역 표준화·리팩터링 | 영역 분할 진행바 + 산출물 매트릭스 |
| 일괄·마이그레이션형 | 다수 파일 일괄 변환 | 대상 목록 + 진행률 + 완료/실패 카운트 |

> 애매하면 사용자에게 한 번 확인. 카탈로그에 없는 작업은 가장 가까운 유형 + 공통 코어만으로. 카탈로그 확장은 별도 결정(`/solo-decision`).

## 5. 공통 코어 (어떤 유형이든 같은 5 요소)

작업의 할 일 목록에서 바로 도출:
1. **진행바** — 전체 진척 % (§7)
2. **단계 목록** — 할 일별 상태 (완료·진행·대기)
3. **현재 위치** — 지금 어느 단계
4. **다음 단계** — 바로 다음 + 선택지
5. **갱신 시점·주체**

## 6. 판단 가이드 (언제 켜나 — 신호 3개 이상)

- 단계(할 일) **3개 이상**
- 여러 **영역·파일**에 걸침
- **장시간·여러 세션** 예상
- **산출물 다수**(보고서·발표자료 등)
- 사용자가 "복잡하다" 언급

→ 3개 이상이면 **"대시보드로 추적하겠습니다" 1줄 고지** 후 생성.

## 7. 진행률 산정 (순수 칸 비율 — 확정)

**완료 칸 ÷ 전체 칸**. 손어림·가중 없음.
- 단순 작업: 완료 할 일 ÷ 전체 할 일
- 단계 매트릭스형: 완료 셀 ÷ 전체 셀 (예: 6/21 = 29%)

> 진행률 밖 성과(인프라·표준 신설 등)는 진행바에 섞지 않고 **별도 성과 배지**로 표기 — 진행바는 정직한 칸 비율만.

## 8. 마커 기반 부분 치환 (구현 핵심 — 전체 재생성 ✗)

미학·정적 영역을 보존하려면 **동적 영역만** 마커 사이를 교체한다:
1. 동적 영역에 앵커 마커 1회: `<!-- SF:키 -->값<!-- /SF:키 -->`
2. 갱신 = 마커 사이만 데이터로 교체 (정적·CSS·여백 불변)
3. **멱등** — 같은 데이터면 결과 동일(diff 없음)

동적 영역 예: 판본·날짜·진행률(바+%)·상세·매트릭스 셀·작업추적·카운트·현재위치·다음단계·footer.
정적(불가침): 제목·골격·`<style>`·범례.

**마커 한계 + 적용 방식** (원 스킬 실측): `<title>` 안·inline `style`(진행바 width)·class 속성엔 HTML 주석을 넣을 수 없다. 그래서 모든 곳에 마커를 일일이 넣는 대신, **대시보드 상단에 「SF:DASHBOARD 갱신 맵」 주석 1개**로 동적/정적 영역 셀렉터(요소 주소)를 명시하고, PL이 그 맵을 보고 **셀렉터로 정밀 치환**한다. 텍스트·블록은 마커 직접 치환, inline style·`<title>`은 셀렉터 기반.

```html
<!-- SF:DASHBOARD 갱신 맵
  동적: <title> | .progress-bar[style*=width] | #pct | #steps | #now | #next | #updated
  정적(불가침): <style> 전체 | .legend | 헤더 골격
-->
```

## 9. 표시 — 로컬 파일 직접(`open`) + 자동 갱신

**원본 HTML을 `open` 으로 기본 브라우저에 직접 띄우되 — 최초 1회만.** 이후 단계 갱신은 **파일만 수정**하고 `open` 을 다시 부르지 않는다 (다시 부르면 같은 `file://` 가 새 탭으로 떠 탭이 누적된다 — 함정 ⑦).

```bash
open "원본경로/dashboard.html"   # macOS — 최초 표시 1회만. 이후는 파일만 갱신
```

**🔄 자동 새로고침 스크립트 (필수 — 모든 대시보드 HTML에 포함)**: 열려 있는 탭이 스스로 최신 파일을 다시 읽도록 `</body>` 직전에 아래를 넣는다. → PL은 **파일만 갱신**하고, 사용자가 그 탭으로 **돌아올 때** 자동 새로고침된다(재표시·새 탭 불필요).

```html
<script>
  // 탭으로 돌아오거나 창에 포커스가 오면 최신 파일로 자동 새로고침 (새 탭 안 띄움)
  var _sfReloading=false; function _sfReload(){ if(_sfReloading) return; _sfReloading=true; location.reload(); }
  document.addEventListener('visibilitychange', function(){ if(document.visibilityState==='visible') _sfReload(); });
  window.addEventListener('focus', _sfReload);
</script>
```

- 원본을 직접 보므로 **항상 최신** (표시본 복사·캐시 우회 0단계), 서버 프로세스 의존 X.
- **갱신 후**: 파일만 Edit → 사용자가 탭 복귀 시 자동 반영(또는 수동 ⌘R). **`open` 재호출 금지**(탭 누적 방지).
- ⚠️ 스크립트가 없던 **기존 탭**은 최초 1회 수동 ⌘R(스크립트 로드) 후부터 자동.

> **폴백 (서버 방식)** — 브라우저 확장·도구로 화면 캡처·DOM 검증이 필요할 때만: 원본 → 임시 폴더 복사 → `python3 -m http.server 8765` → 탭 `?v=N`. 일부 브라우저 확장의 navigate 는 `file://` 을 차단하므로, 확장 캡처가 필요하면 이 서버 폴백을 쓴다.

## 10. 함정 7 종 (원 스킬 실측 이관)

| 함정 | 회피 |
|---|---|
| 한글 경로·괄호 | 경로 항상 따옴표 |
| 표시 방식 | `open "원본"` 로컬 직접 — **최초 1회만** + 자동 새로고침 스크립트 필수(§9) |
| ⑦ open 반복 → 탭 누적 | 갱신은 **파일만 수정**, `open` 재호출 금지. 자동 새로고침 스크립트(visibilitychange/focus) 내장 → 기존 탭이 스스로 최신화(§9) |
| 브라우저 확장 file:// 차단 | 확장 navigate 는 file://→https 강제 변환. 확장 캡처 필요 시 서버 폴백 |

| 날짜 임의 생성 | 호출 시 주입 (재현성) |
| 셀 오매핑 | 산출물 폴더 ↔ 셀 1:1 매핑 고정 |

## 11. 미학 가이드 (자체 완결 — 외부 표준 의존 없음)

대시보드는 단일 HTML 파일로 자체 완결(외부 CSS·CDN·이미지 없음):
- **폰트**: 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans KR", sans-serif`)
- **상태색 3종 고정**: 완료 = 초록(#16a34a 계열) · 진행 = 파랑(#2563eb 계열) · 대기 = 회색(#9ca3af 계열). 진행바·단계 목록·매트릭스 셀이 같은 색 체계를 공유
- **레이아웃**: 상단 헤더(작업명·판본·갱신시각) → 진행바 → 현재 위치/다음 단계 카드 → 단계 목록(또는 유형 특화 매트릭스) → footer(갱신 주체)
- **절제**: 애니메이션·그라데이션 최소, 정보 밀도 우선. 화면 1~2장 안에 전부 들어오게

## 12. 보관 (solo-forge 계약 연계)

- **작업 중 위치(고정)**: `docs/00_관리/dashboards/{작업명}.html` — 작업별 1 파일. 해당 폴더가 없으면 생성한다. (CLAUDE.md 계약의 관리 폴더가 다른 경로면 그 하위 `dashboards/`)
- 작업 종료 시 그대로 보관하고 세션 로그(`docs/00_관리/sessions/`)에서 경로를 링크 — 진행 추적의 **시각 층** 역할. 세션 로그가 "무엇을 했다"의 기록이라면 대시보드는 "어디까지 왔나"의 마지막 스냅샷.
