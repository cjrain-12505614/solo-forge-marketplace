# Figma Make 산출물 검증

Figma Make에서 생성된 디자인 산출물을 검증합니다.

## 사전 요구사항

- Figma Make 사용 여부 확인 (CLAUDE.md에서)
- Figma 파일 URL 준비

## 사용법

```
/review-figma [Figma_URL]
```

### 예시

```
/review-figma https://figma.com/design/abc123/MyDesignFile?node-id=1-2
```

## 실행 단계

### 1. URL 파싱

Figma URL에서 추출:
- fileKey
- nodeId (선택)

### 2. Figma 메타데이터 조회

get_metadata MCP 도구로:
- 페이지 목록
- 프레임 및 컴포넌트 구조
- 변수 정의 (디자인 토큰)

### 3. 검증 항목

```
검증 항목         | 기준 | 확인 | 결과
-----------------|------|------|------
1. 페이지 명명    | 기능별 그룹화 | O | ✓
2. 프레임 표준화  | 크기/레이아웃 일관성 | O | ✓
3. 컴포넌트 재사용| 중복 감소 | O | ✓
4. 디자인 토큰    | 색상/타이포그래피 일관성 | O | ✓
5. 상태 바리에이션| 각 컴포넌트별 4개 이상 | △ | ⚠
6. 접근성         | 명도비/폰트 크기 | △ | ⚠
7. 버전 관리      | 최신 업데이트 | O | ✓
```

### 4. 검증 리포트 생성

저장 경로: `docs/03_설계/figma_review_{날짜}.md`

템플릿:
```markdown
# Figma Make 검증 리포트

**검증 대상**: {파일명}
**검증일**: YYYY-MM-DD
**파일 URL**: [링크]
**검증자**: 준호

## 검증 결과

**종합**: {PASS / CONDITIONAL PASS / FAIL}

## 검증 상세

### 1. 페이지 및 프레임 구조
- ✓ 페이지 명명: {기준} 충족
  - 페이지 목록: [페이지 1, 페이지 2, ...]
- ✓ 프레임 표준화: {확인 내용}

### 2. 컴포넌트 및 재사용성
- {상태} 컴포넌트 수: N개
- {상태} 중복 컴포넌트: M개 발견
- 권장: {개선 사항 또는 "양호"}

### 3. 디자인 토큰
- ✓ 색상 정의: N개
  - 색상표: [색상 1, 색상 2, ...]
- ✓ 타이포그래피: N개
  - 스타일: [스타일 1, 스타일 2, ...]

### 4. 반응형 및 상태 바리에이션
- {상태} 모바일 레이아웃: {확인}
- {상태} 상태 바리에이션: {확인}
  - 권장: {개선 사항 또는 "양호"}

### 5. 접근성 및 품질
- {상태} 명도비 WCAG AA: {확인}
- {상태} 폰트 크기: {확인}
- {상태} 마진/패딩 일관성: {확인}

## 발견 사항

### 주의 (⚠ CONDITIONAL PASS)
1. {이슈 1}: {설명} → {개선안}
2. {이슈 2}: {설명} → {개선안}

### 긴급 (🔴 FAIL)
(해당 사항 없으면 "없음")

## 권장 조치

1. {개선 사항 1}
2. {개선 사항 2}

## 승인 여부

- [ ] PM 승인
- [ ] 개발자 확인
- [ ] 최종 배포 가능

---

**다음 단계**: {다음 작업}
```

### 5. 검증 결과 분류

| 등급 | 조건 | 의미 |
|------|------|------|
| PASS | 모든 항목 ✓ | 즉시 개발 가능 |
| CONDITIONAL PASS | ⚠ 지적사항 있음 | 개선 후 진행 |
| FAIL | 🔴 심각한 결함 | 재설계 필요 |
