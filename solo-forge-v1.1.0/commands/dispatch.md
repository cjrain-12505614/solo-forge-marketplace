# 에이전트 투입

팀원 에이전트에게 작업을 할당합니다.

## 사용법

```
/dispatch [에이전트명] [작업]
```

### 예시

```
/dispatch 하준 경쟁사 3개 조사 및 기능 비교표 작성
/dispatch 연우 로그인 화면 UI 구현
/dispatch 다준 API 통합 테스트 시나리오 작성
```

## 실행 단계

### 1. 팀 정보 읽기

CLAUDE.md Section 2에서 투입할 에이전트명과 역할 확인

### 2. 의사결정 컨텍스트 수집

decision_log.md에서 최근 확정 의사결정 3~5건 수집

### 3. 프롬프트 구성

```
[에이전트명], 당신은 [역할]입니다.

**작업**
{작업 설명}

**컨텍스트**
프로젝트: {프로젝트명}
Phase: {현재 Phase}
최근 의사결정:
  - [D-XXX] {의사결정 1}
  - [D-YYY] {의사결정 2}

**산출물 저장**
{docs 폴더 경로}

**지시사항**
바로 실행하고 완료되면 보고합니다.
```

### 4. Agent 투입

구성된 프롬프트로 Agent 실행

### 5. 완료 보고

```
에이전트 작업 완료.

**투입자**: [에이전트명]
**작업**: [작업 요약]
**산출물**: [생성된 파일 경로]
**주요 결과**: [1~2줄 요약]

검토 후 decision_log.md에 반영하시면 됩니다.
```

## 주의사항

- 에이전트명은 CLAUDE.md Section 2에서 정확히 확인
- 작업은 명확하고 구체적으로 지정
- 기대 산출물(문서/코드 경로)을 명시
- PM이 검토 후 의사결정 기록
