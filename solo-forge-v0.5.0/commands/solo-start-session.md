# 세션 시작

준호가 세션 시작 체크리스트를 실행합니다.

## 실행 단계

1. **CLAUDE.md 읽기** — 프로젝트 정보 및 팀 구성 확인
2. **최신 세션 로그 읽기** — docs/00_관리/sessions 폴더에서 가장 최신 파일 찾기
3. **decision_log.md 읽기** — 미결 의사결정 확인
4. **team_plan.md 읽기** — 팀 상태 및 역할 확인

## 보고 포맷

PM에게 다음 양식으로 보고합니다:

```
PM에게 보고합니다.

**이전 세션**: [날짜] — [주요 완료 사항 1~2줄]
**현재 Phase**: Phase X — [상태]
**미결 의사결정**: [ID 및 항목, 또는 "없음"]
**이번 세션 목표 제안**: [이전 세션 TODO 기반 1~3건]

이 방향으로 진행할까요?
```

### 예시

```
PM에게 보고합니다.

**이전 세션**: 2026-03-22 — 기본 화면 설계 완료, API 스펙 검토중
**현재 Phase**: Phase 2 — 설계 진행중
**미결 의사결정**:
  - D-001: 인증 방식 (OAuth vs JWT)
  - D-002: DB 스키마 최적화
**이번 세션 목표 제안**:
  1. D-001 인증 방식 의사결정 및 확정
  2. API 스펙 최종 검토
  3. DB 마이그레이션 계획 수립

이 방향으로 진행할까요?
```

## 세션 상태 미확인 시

CLAUDE.md나 세션 로그가 없으면:
```
아직 프로젝트가 초기화되지 않았습니다.
/init-project로 프로젝트를 초기 설정해주세요.
```
