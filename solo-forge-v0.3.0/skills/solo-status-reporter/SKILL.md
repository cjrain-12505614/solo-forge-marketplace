# 현황 보고 스킬

## 개요
프로젝트의 현재 상태를 다각적으로 파악하고 종합적인 보고서를 생성합니다.

## 정보 수집 소스

1. **team_plan.md**: 전체 프로젝트 구조, Phase 정의, 일정
2. **최신 세션 로그**: 마지막 작업 내용 및 완료 항목
3. **decision_log.md**: 의사결정 이력, 미결 항목
4. **workspace 폴더 스캔**: 산출물 파일 목록 및 버전
   - docs/00_관리 ~ docs/06_산출물
5. **CLAUDE.md**: 팀 구성, 계약 조건, 현행 규칙

## 보고 형식

## 보고 워크플로우

### 1단계: 정보 수집
```bash
# 필요한 파일들 확인
team_plan.md → Phase 구조 파악
decision_log.md → 의사결정 상태
docs/00_관리/session_YYYY-MM-DD_N.md → 마지막 세션 내용
workspace 폴더 스캔 → 산출물 목록
```

### 2단계: 상태 판정
- **Phase 진행률**: 완료된 산출물 수 / 예상 산출물 수
- **산출물 상태**:
  - ✅ v1.0 이상 (완료)
  - 🔄 v0.x (진행중)
  - ⬜ 파일 없음 (미시작)
- **의사결정 상태**: decision_log.md의 Status 확인

### 3단계: 보고서 생성
- 모든 항목을 위 템플릿에 맞춰 작성
- 명확한 상태 기호 사용 (✅, 🔄, ⬜, ⚠️)
- 구체적인 수치 포함 (진행률, 파일 버전 등)

### 4단계: 저장
- 경로: `docs/00_관리/status_report_YYYY-MM-DD.md`
- 자동 저장 또는 사용자 요청 시 생성

## 사용 예시

### 사용자 요청 1
**사용자**: "현황 알려줘"
→ 전체 현황 보고서 생성 (위 템플릿 전체)

### 사용자 요청 2
**사용자**: "다음에 뭐 해야 해?"
→ "다음 Todo" 섹션 강조 + 블로커 확인

### 사용자 요청 3
**사용자**: "산출물 목록 보여줘"
→ "산출물 현황" 테이블 중심으로 보고

### 사용자 요청 4
**사용자**: "Phase 몇이야?"
→ "현재 상태 요약" + "Phase별 진행률" 섹션

## 주의사항

- 보고서는 **현재 시점 기준**이므로, 실시간 변경사항 반영 필요
- Phase별 진행률은 **예상 진행률**이므로, 의사 결정 변화 시 재계산
- 산출물 버전은 실제 파일 시스템에서 확인 (workspace 폴더)
- 미결 의사결정은 decision_log.md의 최신 상태 기준
