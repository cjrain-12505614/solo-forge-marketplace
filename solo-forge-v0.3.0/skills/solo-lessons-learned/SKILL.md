# 교훈 기록 및 활용 스킬

## 개요
프로젝트 진행 중 발견되는 패턴, 실수, 해결책을 체계적으로 기록하고 향후 작업에 활용합니다.

## 기록 저장소

### 영구 저장소
- **CLAUDE.md Section 11**: 프로젝트 전체에서 반복되는 근본 규칙과 교훈
- 형식: `### 교훈명 (yyyy-mm-dd)`
  - {교훈 내용}
  - **방지/활용 규칙**: {구체적인 규칙}

### 세션별 기록
- 각 세션 로그 끝에 "## 교훈" 섹션 추가
- 형식: `### YYYY-MM-DD`
  - `- {교훈} → {방지/활용 규칙}`

## 기록 트리거

다음 상황에서 반드시 이 스킬을 사용하세요:

1. **PM 피드백**: 작업 결과 수정 요청이 있었을 때
2. **작업 오류**: 잘못된 결과를 발견했을 때
3. **유효 패턴**: 효율적인 해결책이나 프로세스를 발견했을 때
4. **세션 종료**: 주요 배운 점을 정리할 때
5. **사용자 요청**: 사용자가 '교훈 기록해', '다음에 이렇게' 등 명시적으로 요청할 때

## 기록 형식

### 기본 템플릿
```
### YYYY-MM-DD: {간단한 제목}
- 상황: {무엇을 했을 때 어떤 문제가 발생했나}
- 원인: {왜 그렇게 되었나}
- 해결책: {어떻게 해결했나}
- **규칙**: {다음에 반복되지 않도록 할 구체적인 규칙}
```

### 예시
```
### 2026-03-23: 스펙 문서 버전 관리 실수
- 상황: v0.5 → v1.0으로 승격했으나, 검증 체크리스트를 건너뛰었음
- 원인: 빠른 진행을 위해 프로세스를 축약함
- 해결책: 품질 게이트 체크리스트를 먼저 확인하고 결과를 decision_log.md에 기록함
- **규칙**: v0.x → v1.0 승격 시 deliverable-management 스킬의 품질 체크리스트를 반드시 실행. 모든 항목 ✅ 완료 후에만 승격.
```

## 세션 시작 시 활용

1. 세션 시작 전 CLAUDE.md Section 11 읽기
2. 현재 진행 중인 작업에 적용할 규칙 확인
3. 같은 실수를 반복하지 않도록 주의

## 주요 교훈 관리

### 영구 등재 기준
- 프로젝트 전체에 영향을 미치는 패턴
- 반복 가능한 오류 방지 규칙
- 프로세스 표준화 사항

### 세션 기록 기준
- 해당 세션에서만 유효한 학습
- 특정 작업 맥락에 특화된 팁
- 추후 정리 대상 (향후 영구 등재 여부 검토)
