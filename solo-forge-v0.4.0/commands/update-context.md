# /solo-update-context — 작업 완료 후 컨텍스트 동기화

방금 완료한 작업 내용을 프로젝트의 단일 출처(CLAUDE.md 동적 섹션 + 관련 컨텍스트 파일)에 반영한다. `solo-session-protocol`의 세션 종료 동기화와 정합하며, Step/Sprint 도중에도 상태를 최신으로 유지할 때 쓴다.

## 실행 순서

**1단계 — 현재 상태 읽기**
CLAUDE.md와 `$ARGUMENTS`로 지정된 컨텍스트 파일(없으면 프로젝트 관례 경로)을 읽는다. 파일이 없으면 신규 생성 모드로 전환한다.

**2단계 — 변경 내용 파악**
현재 대화에서 완료된 작업을 분석해 아래 중 변경된 것을 뽑는다.
- 새로 구현/변경된 기능·스펙
- 발견되거나 해결된 이슈
- 새로운 의사결정(확정/미결)
- 얻은 교훈·패턴

**3단계 — CLAUDE.md 동적 섹션 동기화**
CLAUDE.md의 아래 섹션을 반영한다(solo-forge 계약 기준). 정적 섹션(1~8)은 건드리지 않는다.
- **Section 9 (확정 의사결정)** — 이번에 확정된 결정을 새 D-ID로 추가
- **Section 10 (미결 의사결정)** — 새로 생긴 미결 추가 / 해소된 항목은 "확정(D-XX)"로 닫기
- **Section 11 (교훈 & 패턴)** — 새 교훈을 L-ID로 추가

> `decision_log.md`가 있으면 Section 9/10과 양방향 동기화한다(`solo-decision-tracker` 규약).

**4단계 — 프로젝트 컨텍스트 파일 갱신**
기능/모듈 컨텍스트 파일이 있으면 아래 섹션을 갱신한다.
```
## 현재 구현 상태
- {기능}: {완료 / 개발 중 / 미착수}

## 최근 변경 이력
- {YYYY-MM-DD}: {변경 내용 한 줄 요약}

## 알려진 이슈
- [ ] {미해결 이슈}
- [x] {해결된 이슈}
```

**5단계 — 완료 보고**
갱신된 파일과 변경 요지를 한두 줄로 요약해 보고한다.
예: "CLAUDE.md §9에 D-12(인증 JWT 확정) 추가, §11에 L-03(마이그레이션 롤백 교훈) 추가. auth 컨텍스트 '기본 CRUD → 완료'로 변경."

## 안전

- 대량/광범위 편집 금지 — 실제 변경된 섹션만 타깃 수정한다.
- 커밋은 이 커맨드의 책임이 아니다(사용자/세션 프로토콜이 수행). git 관련 안전은 프로젝트의 커밋 규약을 따른다.
