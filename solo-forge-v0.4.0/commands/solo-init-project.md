# 프로젝트 초기화

준호(PL)가 새 프로젝트를 초기 설정합니다.

## 사전 검사

CLAUDE.md가 이미 존재하는지 확인 → 있으면 중단하고 "프로젝트가 이미 초기화되어 있습니다."

## 수집 정보

다음 항목을 PM(개발자)에게 물어봅니다:

1. **프로젝트명** (예: "모바일 쇼핑앱")
2. **핵심 목표** (1~2줄)
3. **기술 스택** (언어, 프레임워크, DB, 배포)
4. **예상 기간** (일주일 / 2주 / 1개월 / 기타)
5. **프로젝트 규모** (소: 1~2기능 / 중: 3~5기능 / 대: 6+기능)
6. **Figma Make 사용 여부** (예/아니오)

## 팀 구성 제안

규모별 권장 팀:

```
소: PL 준호 + (필요시 하준/연우)
중: PL 준호 + 기획자(하준) + 개발자(연우) + 검수자(다준)
대: PL 준호 + 기획자(하준) + 개발자(연우) + QA(다준) + 검수자(기타)
```

PM의 확인을 받은 후 CLAUDE.md Section 2에 저장합니다.

## CLAUDE.md 생성

템플릿으로부터 CLAUDE.md 생성:
- Section 1: 프로젝트명, 핵심 목표, 기술 스택, 예상 기간
- Section 2: 팀 구성 및 역할
- Sections 3~10: 기본값 설정
- Section 11: Lessons Log (오늘 날짜 헤더로 시작)

## 초기 문서 작성

**decision_log.md**: 의사결정 이력 (헤더만 작성)
```
# 의사결정 로그

## 확정 의사결정
(항목 없음)

## 미결 의사결정
(항목 없음)
```

**team_plan.md**: 팀 계획
```
# 팀 플랜

| 역할 | 담당자 | 책임 |
|------|--------|------|
| PL | 준호 | 전체 조율, 의사결정 |
| ... | ... | ... |
```
