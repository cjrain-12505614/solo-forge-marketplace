# 교훈 기록

이번 세션이나 작업에서 배운 교훈을 기록합니다.

## 사용법

```
/lesson [교훈 내용]
```

### 예시

```
/lesson 에이전트 투입 시 "바로 실행" 명시 필요 → 지시사항에 "바로 실행하고 보고" 추가
/lesson 의사결정은 선택지별 장단점을 명시해야 PM이 빠르게 판단 가능 → 템플릿에 비교표 추가
/lesson 세션 로그에 산출물 경로를 정확히 기록 → 다음 세션 시작 시 복구 속도 증가
```

## 실행 단계

### 1. CLAUDE.md Section 11 읽기

현재 Lessons Log 구조 확인

### 2. 오늘 날짜 헤더 확인

```
## 2026-03-23

- [기존 교훈들]
```

→ 없으면 새 헤더 생성, 있으면 해당 섹션에 append

### 3. 교훈 기록

형식: `- {교훈} → {방지/활용 규칙}`

예시:
```
- 에이전트 투입 시 "바로 실행" 명시 필요 → 모든 dispatch 프롬프트에 "바로 실행하고 완료 보고" 추가
- 선택지별 비교표 필수 → decision request 템플릿에 비교표 예시 추가
- 체크포인트는 30분마다 → save-session으로 정기적 저장
```

### 4. 완료 보고

```
교훈 기록 완료.

**기록 내용**: [교훈 1줄]
**위치**: CLAUDE.md Section 11 — {오늘 날짜}
**활용**: [다음 작업에 어떻게 적용할지 1줄]
```

## 교훈 기록의 목적

- 같은 실수 반복 방지
- 효율성 개선 규칙 축적
- 팀 공유 지식 확대
- 프로젝트 특수성 반영

## 우수 교훈 예시

| 나쁜 예 | 좋은 예 |
|--------|--------|
| "무언가 잘못됨" | "의사결정 없이 개발 착수 → 설계 승인 전에는 코딩 금지" |
| "더 빠르게 해야 함" | "요구사항 명세 불완전 → feature-spec 템플릿 강화" |
| "협력 문제" | "에이전트에 지시 불명확 → dispatch 프롬프트 체크리스트 추가" |
