# 세션 재개

이전 세션 또는 체크포인트에서 작업을 이어서 시작합니다.

## 실행 단계

### 1. 복구 지점 찾기

우선순위:
1. **체크포인트** — `docs/00_관리/sessions/_checkpoint_*.md` (가장 최신)
2. **세션 로그** — `docs/00_관리/sessions/session_*.md` (가장 최신)

### 2. 컨텍스트 복원

찾은 파일에서:
- 이전 진행 상황 읽기
- 로드된 컨텍스트 목록 확인
- 다음 작업 확인

### 3. 재개 보고

```
세션 재개 완료.

**복구 지점**: [체크포인트 또는 세션 로그 파일명]
**이전 진행**: [현재 진행 상황 1~2줄]
**로드된 컨텍스트**:
  - [문서 1]
  - [문서 2]
**즉시 다음 작업**: [다음 작업]

계속 진행할까요?
```

### 예시

```
세션 재개 완료.

**복구 지점**: _checkpoint_2026-03-23_14-30.md
**이전 진행**: API 스펙 검토 중, 인증 부분까지 완료
**로드된 컨텍스트**:
  - docs/02_기획/api_spec.md
  - decision_log.md
  - team_plan.md
**즉시 다음 작업**: 데이터베이스 스펙 검토 및 확정

계속 진행할까요?
```

## 복구 지점 없음

체크포인트와 세션 로그가 모두 없으면:
```
복구할 이전 세션/체크포인트가 없습니다.
/start-session으로 새 세션을 시작해주세요.
```
