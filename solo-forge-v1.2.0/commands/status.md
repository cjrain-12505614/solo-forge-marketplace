# 프로젝트 현황

현재 프로젝트의 전체 상황을 한눈에 파악합니다.

## 실행 단계

### 1. 정보 수집

- **team_plan.md**: Phase 진행률, 팀 상태
- **최신 세션 로그**: 가장 최근 완료 사항
- **decision_log.md**: 미결 의사결정 현황
- **docs 폴더 스캔**: 산출물 파일 목록 및 갱신 시간

### 2. 현황 보고

```
현황 보고 — YYYY-MM-DD HH:mm

## Phase 진행률
Phase X / Y 단계 (예상 진행률 XX%)

## 산출물 현황
| 문서 | 상태 | 최종 갱신 |
|------|------|---------|
| docs/01_기초/... | ✓ 완료 | YYYY-MM-DD |
| docs/02_기획/... | ⟳ 검토중 | YYYY-MM-DD |
| ... | | |

## 미결 의사결정
- [D-XXX] [주제]: 검토중
- [D-YYY] [주제]: PM 승인 대기

(없으면 "없음")

## 다음 TODO
1. [작업 1 — Phase X]
2. [작업 2 — Phase X]
3. [작업 3 — Phase X]

## 위험 요소
- [있으면 기록, 없으면 "없음"]
```

### 3. 예시

```
현황 보고 — 2026-03-23 15:30

## Phase 진행률
Phase 2 / 5 단계 (예상 진행률 40%)

## 산출물 현황
| 문서 | 상태 | 최종 갱신 |
|------|------|---------|
| 기본 요구사항 | ✓ 완료 | 2026-03-22 |
| 기능 명세 | ✓ 완료 | 2026-03-22 |
| API 스펙 | ⟳ 검토중 | 2026-03-23 |
| DB 스키마 | ▯ 미작성 | - |

## 미결 의사결정
- [D-001] 인증 방식: 검토중
- [D-002] 캐싱 전략: PM 승인 대기

## 다음 TODO
1. API 스펙 최종 검토 — Phase 2
2. DB 스키마 설계 — Phase 3
3. 개발 환경 구성 — Phase 4

## 위험 요소
없음
```
