# 세션 프로토콜 스킬

이 스킬은 프로젝트 세션의 시작, 진행 중, 종료를 관리합니다. 세션의 컨텍스트 복원과 진행 상황 저장을 통해 프로젝트 연속성을 보장합니다.

## 1. 세션 시작 프로토콜 (5단계)

세션이 시작될 때마다 다음 순서로 실행합니다.

### Step 1: CLAUDE.md 읽기

```
경로: {프로젝트_루트}/CLAUDE.md
확인 항목:
- Section 1: 프로젝트 개요 ✓
- Section 2: 팀 구성 (에이전트 역할 확인) ✓
- Section 3: 워크스페이스 구조 ✓
- Section 5: 의사결정 프로토콜 ✓
- Section 6: 역할 분담 (Cowork ↔ Claude Code) ✓
- Section 7: 실행 규칙 ✓
```

### Step 2: 최신 세션 로그 로드

```
경로: docs/00_관리/sessions/
작업:
1. 가장 최신 session_YYYY-MM-DD_N.md 파일 읽기
2. "이전 세션 요약" 섹션 추출
3. "완료/미완료 태스크" 확인
4. "다음 세션 TODO" 로드 (가장 중요함)
```

### Step 3: decision_log.md 확인

```
경로: docs/00_관리/decisions/decision_log.md
작업:
1. Section 10 (미결 의사결정) 확인
2. 이번 세션에 영향을 미치는 미결 사항 파악
3. 진행 중인 의사결정 업데이트 필요 여부 판단
```

### Step 4: team_plan.md 확인

```
경로: docs/00_관리/team_plan.md
작업:
1. 현재 Phase 확인
2. 이번 Phase의 목표와 태스크 로드
3. 완료율(%) 파악
4. 이번 세션 투입할 에이전트 및 태스크 결정
```

### Step 5: PM에게 세션 시작 보고

**보고 형식**:

```
PM, 세션을 시작합니다.

## 이전 세션 요약
- 완료된 산출물: {산출물 리스트}
- 미완료 태스크: {태스크 리스트}
- 주요 결정사항: {의사결정 리스트}

## 현재 상태
- **현재 Phase**: {Phase명} (진행률 XX%)
- **담당 에이전트**: {이번 세션 투입 에이전트}

## 미결 의사결정
- 🔴 {D-번호}: {항목} (PM 확인 필요)
- 🟡 {D-번호}: {항목} (PL 판단 필요)

## 이번 세션 목표 (제안)
1. {목표 1}: {담당 에이전트}
2. {목표 2}: {담당 에이전트}
3. {목표 3}: {담당 에이전트}

이 계획으로 진행해도 될까요?
```

### Step 2: decision_log.md 업데이트

```
경로: docs/00_관리/decisions/decision_log.md

작업:
1. 세션 로그의 "의사결정" 섹션에서 결정된 항목 추출
2. 각 D-번호별로 decision_log.md 테이블 업데이트
3. "D별 상세 기록" 섹션에 새로운 결정 내용 추가
4. Section 9 (확정) 또는 Section 10 (미결)로 분류
```

**업데이트 형식**:

```markdown
### D-XX: {항목}

**날짜**: YYYY-MM-DD
**심각도**: 🔴/🟡/🟢
**배경**: {왜 필요했는지}
**선택지**:
- A) {선택지 1} — {장점}, {단점}
- B) {선택지 2} — {장점}, {단점}

**결정**: {선택된 안}
**근거**: {결정 근거}
**결정자**: {PM/PL/에이전트명}
**영향 범위**: {영향을 받는 영역들}
**반영 세션**: session_YYYY-MM-DD_N
```

### Step 3: CLAUDE.md Section 9, 10 동기화

```
작업:
1. Section 9 (확정 의사결정): decision_log.md에서 상태 = "완료"인 항목
2. Section 10 (미결 의사결정): decision_log.md에서 상태 = "진행 중"인 항목
3. 테이블 형식 업데이트
```

### Step 4: team_plan.md 업데이트

```
경로: docs/00_관리/team_plan.md

작업:
1. 완료된 태스크 체크 표시
2. Phase 완료율 재계산
3. 다음 Phase로의 진행 여부 판단
4. 에이전트 배치 계획 업데이트
```

**형식**:

```markdown
## Phase 2: 설계 (진행률 45%)

### 산출물
- [x] 시스템 아키텍처 v1.0 (session_2026-03-21_2)
- [ ] DB 스키마 설계 (대기 중)
- [ ] API 명세 v1.0 (진행 중 - 수진)

### 태스크 진행
- [x] 기술 검토 (2026-03-20)
- [ ] 디자인 검토 (예상: 2026-03-24)
- [ ] 구현 준비 회의 (예상: 2026-03-25)
```

### Step 5: CLAUDE.md Section 11 업데이트

```
경로: CLAUDE.md - Section 11: 교훈 & 패턴

작업:
1. 세션 로그의 "배운 점" 섹션에서 추출
2. 재사용 가능한 패턴으로 정규화
3. 자주 하는 실수 패턴에 해결책 기록
4. 효과적인 패턴에 사용 시점 기록

형식:
- {패턴명}: {원인/효과} → {해결책/사용 시점}
```

## 3. Checkpoint (세션 중간 저장)

세션이 진행 중일 때 중간 저장이 필요한 경우:

```
명령어: "체크포인트 저장", "임시 저장", "중간 진행 상황 저장"

작업:
1. 현재까지의 완료 태스크 기록
2. 현재 에이전트 투입 상태 메모
3. 다음 태스크 준비 상황 기록
4. 컨텍스트 파일 로드 상태 저장

출력:
- checkpoint_YYYY-MM-DD_N_interim.md 파일 생성
- "저장되었습니다. 이어서 진행할 준비가 되었습니다." 보고
```

## 5. 세션 기록 관리

### 파일명 규칙

```
session_{YYYY-MM-DD}_{N}.md

예시:
- session_2026-03-23_1.md (첫 번째 세션)
- session_2026-03-23_2.md (같은 날 두 번째 세션)
- session_2026-03-24_1.md (다음 날 첫 번째 세션)
```

### 보관 위치

```
docs/00_관리/sessions/
└── session_2026-03-23_1.md
└── session_2026-03-23_2.md
└── session_2026-03-24_1.md
└── ...
```

### 세션 히스토리 추적

CLAUDE.md Section 4에서 유지:

```markdown
## 4. 세션 관리 & 히스토리

**현재 Phase**: 기획 진행 중
**총 세션 수**: 3

**최근 세션**:
- session_2026-03-24_1: 요구사항 명세 v1.0 작성 완료
- session_2026-03-23_2: 기술 조사 및 아키텍처 방향 결정
- session_2026-03-23_1: 프로젝트 초기 설정 및 팀 구성

자세한 히스토리는 `docs/00_관리/sessions/` 참고.
```

## 7. 트러블슈팅

```
해결책:

2. 최근 CLAUDE.md Section 4에서 마지막 세션 번호 확인

```

```
해결책:
1. docs/00_관리/_CLAUDE.md (백업) 확인

2. 🟡 심각도 항목들은 PL이 빠르게 판단
3. 🟢 심각도 항목들은 담당 에이전트가 자율 처리
4. decision_log.md에서 "대기 중인 의사결정" 섹션 추가
```

## 8. 세션 프로토콜 실행 팁

**팁 1: "다음 세션 TODO"가 가장 중요**
- 이 섹션이 명확하면 다음 세션 시작이 빠름
- 항상 우선순위와 담당자를 포함
- 예상 소요 시간도 기록하면 좋음

**팁 2: decision_log.md를 세션 초반에 한 번 더 읽기**
- 세션 중 새로운 의사결정이 나올 가능성
- 의사결정 시점을 정확히 기록

**팁 3: Phase 진행률을 정확히 계산**
- 완료 태스크 / 전체 태스크 × 100%
- team_plan.md에서 다음 Phase 전환 시기 판단

**팁 4: 세션 노트는 실시간 작성**
- 종료 후 정리하지 말고 진행 중 계속 추가
- 나중에 보고서 작성할 때 참고

**팁 5: PM과의 커뮤니케이션은 정형화된 형식 사용**
- 시작 보고, 종료 보고 형식 일관성 유지
- PM이 빠르게 파악 가능
